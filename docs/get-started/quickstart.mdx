Let's build a simple text processing pipeline to understand the core workflow.

## 1. Define Nodes

Nodes are just Python functions decorated with `@node`. They declare their inputs and outputs.

```python
from hypernodes import node

@node(output_name="cleaned_text")
def clean_text(passage: str) -> str:
    return passage.strip().lower()

@node(output_name="word_count")
def count_words(cleaned_text: str) -> int:
    # Note: 'cleaned_text' matches the output of the previous node
    return len(cleaned_text.split())
```

## 2. Build & Run Pipeline

Create a pipeline and run it with a single input.

```python
from hypernodes import Pipeline

# Dependencies are automatically resolved!
pipeline = Pipeline(nodes=[clean_text, count_words])

# Run with a single input
result = pipeline.run(inputs={"passage": "  Hello HyperNodes  "})
print(result)
# Output: {'cleaned_text': 'hello hypernodes', 'word_count': 2}
```

## 3. Add Caching

This is where HyperNodes shines. Add a `DiskCache` to persist results.

```python
from hypernodes import SeqEngine, DiskCache

# Configure engine with caching
engine = SeqEngine(cache=DiskCache(path=".cache"))
pipeline = Pipeline(nodes=[clean_text, count_words], engine=engine)

# First run: Executes nodes
pipeline.run(inputs={"passage": "Hello World"})

# Second run: INSTANT cache hit!
pipeline.run(inputs={"passage": "Hello World"})
```

## 4. Scale with `.map()`

Process thousands of items without changing your node logic.

```python
# Process multiple items
results = pipeline.map(
    inputs={"passage": ["Item 1", "Item 2", "Item 3"]},
    map_over="passage"
)
```

Each item is cached independently. If you add one new item to the list, **only that new item is processed**.
